## 四、重点问题（TOP 5）

### 1. 收藏功能完全失效（BUG-001）
- **问题描述**: 收藏按钮UI状态变化正常（高亮/实心切换），但收藏数据未持久化到后端。/favorites 页面显示"加载失败，请检查网络后重试"，所有收藏内容丢失
- **影响范围**: 帖子收藏 + 商品收藏，涉及4个测试用例失败，是失败最多的单一问题
- **建议方案**: 排查 FavoriteController 的 API 路径（已知存在 /api/favorites 与 /api/v2/favorites 不统一问题），检查前端 api.js 中收藏相关接口调用路径和参数格式，确认后端接口是否正常响应

### 2. 发布商品核心流程受阻（BUG-002）
- **问题描述**: 只能上传一张图片（应支持多图）；填写校区和交易地点字段后提交失败
- **影响范围**: 商品发布是平台核心功能，直接影响卖家体验
- **建议方案**: 多图上传问题排查 ImageUploader 组件的文件列表管理和后端分片上传接口；校区/地点字段问题排查 CreateProduct.vue 提交时的字段映射和后端 Product 实体字段对应关系

### 3. 路由守卫安全漏洞（BUG-003 + BUG-010）
- **问题描述**: 未登录可直接访问 /products/create、/community/posts/create 以及所有组织页面（/orgs/create、/orgs/my、/orgs/discover、/orgs/:id）
- **影响范围**: 安全问题，未授权用户可进入发布和管理页面
- **建议方案**: 在 router/index.js 的路由守卫白名单中移除这些路由，或将它们加入需要认证的路由列表

### 4. 聊天实时性和消息感知缺失（BUG-005 + BUG-006）
- **问题描述**: WebSocket消息非实时推送，需手动刷新；会话列表无未读消息数，TabBar/Header无未读徽章
- **影响范围**: 即时聊天体验严重受损，用户无法感知新消息
- **建议方案**: 排查 WebSocket 连接是否正常建立和消息监听；检查 notification store 的未读数轮询逻辑是否正常工作；确认后端 WebSocket Handler 的消息推送机制

### 5. 移动端输入框被键盘遮挡（BUG-012）
- **问题描述**: 帖子详情页评论输入框和聊天页输入框在虚拟键盘弹出时被遮挡，用户看不到自己输入的内容
- **影响范围**: 移动端核心交互场景，影响评论和聊天功能可用性
- **建议方案**: 为输入区域添加 `position: sticky` 或在键盘弹出时动态调整滚动位置；聊天页可使用 `visualViewport` API 监听键盘变化

## 五、结论与建议

### 整体评价

项目核心框架稳定，基础功能（登录/注册/信息流/商品浏览/帖子浏览/评论/关注/组织创建）运行正常，PC端通过率81.5%，移动端77.8%。主要问题集中在：

1. **收藏功能完全失效**是最严重的问题，影响多个模块的测试结果
2. **安全相关**的路由守卫遗漏需要尽快修复
3. **移动端适配**存在输入框遮挡和触摸漂移等影响可用性的问题
4. **搜索模块**通过率最低（62.5%），标签搜索、URL参数搜索、关键词高亮均未实现

### 修复优先级建议

| 批次 | 内容 | 涉及缺陷 |
|------|------|---------|
| 批次1 | P0安全+阻断 | BUG-001(收藏), BUG-002(发布商品), BUG-003(路由守卫) |
| 批次2 | P1安全+聊天 | BUG-010(组织路由守卫), BUG-005(消息实时), BUG-006(未读数) |
| 批次3 | P1功能修复 | BUG-004(未登录操作), BUG-007~009, BUG-011~015 |
| 批次4 | P2体验优化 | BUG-016~023 |

### 不作为缺陷记录的项目

- 注册后自动登录跳转首页（T01-01）：经确认属于设计选择，非缺陷
- 取消报名功能（T06-05）：经确认不需要此功能
- 收货地址管理（T09-11）：经确认不需要此功能
- WebSocket连接测试（T05-04）：黑盒测试无法验证，留待白盒测试
