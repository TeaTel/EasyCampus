# 易校 EasyCampus 集成黑盒测试流程

> 生成时间: 2026-06-07
> 测试站点: <https://c2cmarket.store>
> 测试范围: PC端 + 移动端 全面集成黑盒测试

***

## 一、测试概述

### 1.1 测试目标

对易校校园图文交流平台进行全面的集成黑盒测试，验证以下方面：

| 维度    | 说明                                         |
| ----- | ------------------------------------------ |
| 功能正确性 | 各功能模块是否按预期工作                               |
| 响应式适配 | PC端(1920/1440/1366)和移动端(375/390/414)的布局与交互 |
| 用户体验  | 加载态、空状态、错误状态、交互反馈                          |
| 边界健壮性 | 网络异常、Token过期、并发操作、极限输入                     |
| 导航完整性 | 所有路由可达、页面间跳转正确、TabBar正确显示/隐藏               |

### 1.2 测试环境要求

| 环境    | PC端                                 | 移动端                                                            |
| ----- | ----------------------------------- | -------------------------------------------------------------- |
| 设备    | Mac/Windows 笔记本或台式机                 | iPhone/Android 手机                                              |
| 浏览器   | Chrome 最新版 + Firefox 最新版            | Chrome 移动版 + Safari 移动版                                        |
| 分辨率   | 1920×1080 / 1440×900 / 1366×768     | 375×812 (iPhone X) / 390×844 (iPhone 14) / 414×896 (iPhone 11) |
| 开发者工具 | 必须开启，用于模拟移动端、网络限速、查看Console/Network | -                                                              |
| 网络    | 正常WiFi(基线) + 限速Slow 3G(异常测试)        | 4G / WiFi                                                      |
| 测试账号  | 需准备2个已注册账号(用于聊天/关注等交互测试)            | 同PC端                                                           |

### 1.3 测试账号准备

测试前需确保以下账号可用（自行注册或使用已有账号）：

| 账号  | 用途                     |
| --- | ---------------------- |
| 账号A | 主测试账号，执行大部分操作          |
| 账号B | 辅助测试账号，用于聊天/关注/评论等交互验证 |
| 未登录 | 验证路由守卫和公开页面可访问性        |

### T01 用户认证模块（15条）

#### T01-01 正常注册

- **优先级**: P0
- **前置条件**: 未登录，准备一个未注册的用户名和邮箱
- **测试步骤**:
  1. 访问 `/register`
  2. 填写用户名、昵称、邮箱、密码
  3. 勾选用户协议
  4. 点击注册按钮
- **预期结果**: 注册成功，跳转到登录页，URL携带 `?registered=true&username=xxx`
- **PC端结果**: 直接注册成功，但是直接跳转到了主页，并没有跳转到登录页，url也没携带`?registered=true&username=xxx`
- **移动端结果**: 同上

#### T01-02 重复用户名注册

- **优先级**: P1
- **前置条件**: 使用已存在的用户名
- **测试步骤**:
  1. 访问 `/register`
  2. 填写已存在的用户名，其余正常填写
  3. 点击注册
- **预期结果**: 显示错误提示"用户名已存在"或类似提示，不跳转
- **PC端结果**:与预期一致
- **移动端结果**: 与预期一致

- **复现步骤**: 精确的操作步骤
